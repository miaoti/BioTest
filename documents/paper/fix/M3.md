## 3. Power analysis at n=10

For n₁ = n₂ = 10 the U-distribution under H₀ has 184 756 arrangements
(C(20, 10)). The exact two-sided MWU p-value at every Â₁₂ from 0 to 0.5
in 0.05 steps:

| Â₁₂ | min(U) | exact p (two-sided) | Holm × 4 (worst) | Holm × 3 | Holm × 2 | Holm × 1 (best) |
|:--:|:--:|:--:|:--:|:--:|:--:|:--:|
| 0.00 |  0 | 1.08 × 10⁻⁵ | 4.30 × 10⁻⁵ | 3.20 × 10⁻⁵ | 2.20 × 10⁻⁵ | 1.10 × 10⁻⁵ |
| 0.05 |  5 | 2.06 × 10⁻⁴ | 8.20 × 10⁻⁴ | 6.20 × 10⁻⁴ | 4.10 × 10⁻⁴ | 2.10 × 10⁻⁴ |
| 0.10 | 10 | 1.51 × 10⁻³ | **6.02 × 10⁻³**  ← passes α=0.05 | 4.51 × 10⁻³ | 3.01 × 10⁻³ | 1.51 × 10⁻³ |
| 0.15 | 15 | 6.84 × 10⁻³ | 2.74 × 10⁻² | 2.05 × 10⁻² | 1.37 × 10⁻² | 6.84 × 10⁻³ |
| 0.20 | 20 | 2.32 × 10⁻² | 9.29 × 10⁻² | 6.97 × 10⁻² | 4.65 × 10⁻² | 2.32 × 10⁻² |
| 0.25 | 25 | 6.30 × 10⁻² | 2.52 × 10⁻¹ | 1.89 × 10⁻¹ | 1.26 × 10⁻¹ | 6.30 × 10⁻² |
| 0.30 | 30 | 1.43 × 10⁻¹ | (caps at 1) | 4.29 × 10⁻¹ | 2.86 × 10⁻¹ | 1.43 × 10⁻¹ |

Rule for clearing Holm–Bonferroni at family-wise α = 0.05 with k=4
cells: the worst-rank raw p must be ≤ 0.0125, equivalently Â₁₂ ≤ 0.10
(or ≥ 0.90) at n=10,n=10. The minimum-effect floor for Holm-clearance
at this n is therefore Â₁₂ ≤ 0.10, which corresponds to "BioTest below
baseline on at least 90 of 100 rep-pairings."

Three of the four close-margin cells already show **Â₁₂ = 0** at n=4
(BioTest below baseline on every rep-pairing). If that ordinal pattern
persists at n=10, those cells project to:

- raw exact p ≈ 1.08 × 10⁻⁵
- Holm-Bonferroni-corrected p ≈ 4.30 × 10⁻⁵ (worst-rank, ×4)

That comfortably clears family-wise α = 0.05 and α = 0.001. The
remaining cell (`vcfpy`, Â₁₂ = 0.25 at n=4 because four of sixteen
n=4 rep-pairings have BioTest ≥ Atheris) is the cell where the
boost might fail to deliver an inferential gap: at the same Â₁₂
the n=10 raw p is 0.063, which neither raw nor Holm clears at α=0.05.

### 4.3 Aggregator (n=10 statistics from per-rep JSONs)

```python
# compares/scripts/m3_aggregate_n10.py — new
import json, os, statistics, sys
from itertools import combinations
from math import comb
from collections import Counter

CELLS = [
    ("htsjdk_vcf",  "biotest_run1_rep_*/htsjdk_vcf",  "jazzer_4rep/rep_*/htsjdk_vcf"),
    ("htsjdk_sam",  "biotest_run1_rep_*/htsjdk_sam",  "jazzer_4rep/rep_*/htsjdk_sam"),
    ("vcfpy",       "biotest_run1_rep_*/vcfpy",        "atheris/vcfpy_runs/run_*"),
    ("noodles",     "biotest_run1_rep_*/noodles",      "cargo_fuzz/noodles/run_*"),
]

def per_rep_scores(glob_pattern, n_expected=10):
    import glob
    scores = []
    for p in sorted(glob.glob(f"compares/results/mutation/{glob_pattern}/summary.json")):
        d = json.load(open(p))
        if d.get("reachable", 0) > 0:
            scores.append(d["killed"] / d["reachable"])
    return scores

def mwu_two_sided_exact_p(s_a, s_b):
    n_a, n_b = len(s_a), len(s_b)
    pooled = sorted(s_a + s_b)
    rank = {}
    i = 0
    while i < len(pooled):
        j = i
        while j+1 < len(pooled) and pooled[j+1] == pooled[i]:
            j += 1
        avg = (i + j) / 2 + 1
        for k in range(i, j+1):
            rank[(pooled[k], k)] = avg
        i = j + 1
    used = [False]*len(pooled); R_a = 0
    for x in s_a:
        for k in range(len(pooled)):
            if not used[k] and pooled[k] == x:
                R_a += rank[(pooled[k], k)]; used[k] = True; break
    obs_U = min(R_a - n_a*(n_a+1)/2, n_a*n_b - (R_a - n_a*(n_a+1)/2))
    total = comb(n_a + n_b, n_a); extreme = 0
    for combo in combinations(range(n_a + n_b), n_a):
        R = sum(rank[(pooled[k], k)] for k in combo)
        U = min(R - n_a*(n_a+1)/2, n_a*n_b - (R - n_a*(n_a+1)/2))
        if U <= obs_U: extreme += 1
    return extreme / total

def vargha_a12(s_a, s_b):
    gt = sum(1 for x in s_a for y in s_b if x > y)
    eq = sum(1 for x in s_a for y in s_b if x == y)
    return (gt + 0.5 * eq) / (len(s_a) * len(s_b))

results = []
for cell, bt_glob, base_glob in CELLS:
    bt = per_rep_scores(bt_glob); base = per_rep_scores(base_glob)
    a12 = vargha_a12(bt, base)
    p   = mwu_two_sided_exact_p(bt, base)
    results.append((cell, a12, p, bt, base))

# Holm-Bonferroni across 4 cells
results_sorted = sorted(results, key=lambda r: r[2])
m = len(results_sorted)
for rank_i, (cell, a12, p, bt, base) in enumerate(results_sorted, start=1):
    p_holm = min(1.0, p * (m - rank_i + 1))
    print(f"{cell:>14}  n_BT={len(bt)}/n_base={len(base)}  A12={a12:.4f}  raw_p={p:.5f}  Holm_p={p_holm:.5f}")
```

## 8. Limitations and methodology notes

- The power analysis assumes no ties in the rank assignment. The n=4
  vcfpy data has no ties (all eight scores are distinct), and with
  continuous-valued mutation scores the assumption holds at n=10 too
  with overwhelming probability.
- We pre-register the six BioTest Rank-13 seeds
  (301, 401, 501, 601, 701, 801) and the family-wise α = 0.05 on the
  four close-margin cells. Pre-registration is recorded in
  `compares/scripts/run_mutation_boost_m3.py`.
- The htsjdk/SAM Jazzer baseline is n=2/4 at the existing 4-rep level
  due to a documented PIT anomaly (`compares/results/jazzer_4rep_baseline.md`
  §"Known issues"). The boost runs six additional Jazzer × SAM reps;
  if the PIT anomaly persists at the same ~50% rate the SAM baseline
  lands at n ≈ 5 of 10, and the exact-p calculation accepts unbalanced
  n. If the anomaly disappears (root cause unknown but flagged for
  `--verbose=true` follow-up), n_base = 8 of 10 is achievable.
- **Atheris × vcfpy budget.** The existing 4-rep Atheris baseline at
  `atheris/vcfpy_runs/run_{0..3}/` uses a 7200 s coverage budget per
  rep. We retain the same budget for the boost reps to avoid
  protocol asymmetry; per-rep wall is ~135 min (7200 s atheris +
  ~17 min mutmut), so 6 boost reps ≈ 13.5 wall-hours and likely
  cannot all complete in a single session. Whatever subset completes
  by the time the paper goes to typesetting is reported in §9; the
  exact-p calculation accepts unbalanced n.

### 9.2 Notes on the cargo-fuzz boost

- **run_06 baseline failure.** cargo-mutants requires the baseline
  cargo test (which feeds the corpus through the noodles parser
  and asserts non-error responses) to pass before mutation testing
  can begin. For run_06's 200-file random sample, at least one
  corpus file made the noodles baseline assert. cargo-mutants
  exited rc=4 after 17 s. The aggregator filters reachable=0
  entries; n_base for noodles is 9, not 10.
- **run_07 outlier (3.34 % vs cluster 9.0–10.0 %).** Investigated:
  cargo-mutants for run_07 exited cleanly at rc=0 with reach=299
  but only 10 mutants killed (vs cluster ~28). The corpus dropped
  the per-mutant test from "executes a parser branch that the
  mutated code is on" to "executes only an early-exit branch
  before reaching the mutated code" for ~18 mutants on this
  particular sample. We retain the rep — it is a legitimate
  cargo-fuzz × noodles outcome under the same recipe — but it
  pulls the n=10 std from ~0.30 (existing 4-rep) up to 1.91. The
  mean shifts only slightly because the outlier is one of nine
  values (8.74 vs 9.49 without it). Â₁₂ on this cell rose from
  0.000 (n=4) to 0.111 (n=10): the inversions all come from
  cargo-fuzz run_07 (3.34 %) sitting below every BioTest rep, so
  10 of 90 pairings (one per BioTest rep, all involving run_07)
  have BioTest above cargo-fuzz. The Holm-corrected p still
  passes at 0.003 under this revised pattern.

### 9.3 Path / encoding fixes applied during the boost run

Three small bugs surfaced in the §4 reproducer scripts during the
run and were fixed in-session; the scripts now run cleanly. Recording
them so the next operator does not rediscover:

1. **`coverage_sampler.py` for atheris must run on the host, not
   inside `biotest-bench:latest`.** The internal `_run_atheris_rep`
   helper invokes `docker run biotest-bench:latest …` as a subprocess
   to spin up the Python 3.11 atheris-venv; if `coverage_sampler.py`
   is itself wrapped in `docker run …`, the inner `docker` invocation
   fails with `FileNotFoundError: 'docker'` because the bench image
   does not contain a docker client. Fix: drop the outer Docker
   wrapper (now applied in `run_atheris_boost_m3.sh`).
2. **coverage_sampler.py with `--reps 1 --out X` writes to
   `X/run_0/corpus`, not `X/corpus`.** The mutation step's
   `--corpus` argument must therefore be `$COV_OUT/run_0/corpus`,
   not `$COV_OUT/corpus`. Both `run_atheris_boost_m3.sh` and
   `run_cargo_fuzz_boost_m3.sh` were updated.
3. **`mutation_driver.py` prints Unicode arrows (U+2192 "→") that
   fail under Windows cp1252 stdout redirection.** Set
   `PYTHONIOENCODING=utf-8` before invoking `mutation_driver.py`
   from a wrapper that redirects stdout to a log file. Both
   atheris and cargo-fuzz boost scripts updated.
4. **vcfpy mutmut summary.json reports `killed=0` when the mutmut
   wrapper does not parse mutmut 3.x's reduced text report
   correctly.** Authoritative kill counts are recoverable from
   `mutants/vcfpy/*.py.meta`'s `exit_code_by_key` dict
   (exit 0 = survived, exit 1 = killed, exit 33 = no_tests). For
   the BioTest pipeline, `run_mutation_run1.py::read_summary`
   automatically calls `compares/scripts/mutation/rederive_from_meta.py`
   when this happens. For the atheris vcfpy_runs/ layout the
   rederive script's expected `cell_dir/mutants/<package>/` layout
   does not match (mutants live at
   `cell_dir/vcfpy/mutants/vcfpy/`), so a small adapter script is
   needed (run inline in §9 above) — this is a reproducibility
   gap worth fixing in `rederive_from_meta.py` before the next
   atheris boost.
