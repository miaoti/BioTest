# PIT — Java mutation testing for htsjdk

PIT (pitest) injects bytecode-level mutations into a Java project and reports which test cases "kill" which mutants. Used here to score the test suites generated by BioTest, EvoSuite, Randoop, and Pure Random against **htsjdk**.

## Version

`org.pitest:pitest-command-line:1.15.3` (or the 1.16.x line if released by run time).

## Scope restriction (critical)

Per `compares/DESIGN.md` §3.3, mutation targets are restricted to htsjdk's VCF and SAM I/O code:

```
targetClasses = [
  "htsjdk.variant.vcf.*",
  "htsjdk.variant.vcf2.*",      # if present in the pinned htsjdk version
  "htsjdk.samtools.*",
  "htsjdk.samtools.util.*"      # only the parts that touch file I/O
]
```

Mutating the entire htsjdk codebase is wasteful and dilutes the score — parsers for BAM, CRAM, and unrelated utilities are out of scope.

## Mutators

Default group: `CONDITIONALS_BOUNDARY`, `INCREMENTS`, `MATH`, `NEGATE_CONDITIONALS`, `RETURN_VALS`, `VOID_METHOD_CALLS`. Richer groups (STRONGER, ALL) can be enabled later if score differentiation is insufficient.

## Invocation (planned Gradle task, not yet added to htsjdk build)

```
java -cp <pit JAR plus htsjdk classpath plus test JAR> \
  org.pitest.mutationtest.commandline.MutationCoverageReport \
  --sourceDirs <htsjdk src/main/java> \
  --classPath <same> \
  --targetClasses 'htsjdk.variant.vcf.*,htsjdk.samtools.*' \
  --targetTests '<generated-suite-classes>' \
  --reportDir compares/results/pit/<tool_name>/ \
  --outputFormats HTML,XML
```

The XML report is parsed by `compares/scripts/mutation_driver.py` and aggregated into the shared JSON results schema defined in `DESIGN.md` §4.3.

## Notes

- BioTest generates *file-level* test cases, not JUnit classes. To run PIT against BioTest's suite we wrap each generated VCF/SAM file in a JUnit test that parses it with the SUT and asserts against the canonical-JSON baseline captured during Phase C — i.e., the test *is* the metamorphic/differential check. A small adapter will be needed; see `compares/scripts/mutation_driver.py` (TBD).
- PIT has line-level coverage filtering built in — it skips mutants in lines no test executes. That aligns with the "reachable mutants" definition in DESIGN.md §3.3.
